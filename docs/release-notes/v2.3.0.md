# Release Notes v2.3.0

## 🚀 New Features

### Configurable request timeout
- Added `timeout` parameter to `ConnectionPool` (defaults to `5.0` seconds)
- Previously hardcoded, the timeout can now be set at initialization: `ConnectionPool(timeout=30.0)`

### Graceful pool shutdown
- `await pool.shutdown()` closes all connections and releases resources
- Use this when shutting down your application to avoid leaking connections

## 🔧 Improvements

### Ephemeral client lifecycle for per-call overrides
- Per-call `retry`/`backoff` overrides now properly open and close an ephemeral HTTP client for each request
- The ephemeral client is always closed after the request, even on errors, preventing connection leaks

> **Note:** Per-call `retry` or `backoff` overrides create and tear down a connection (TCP + TLS handshake) for every request. Because an email send is a lightweight operation — a small JSON payload answered with a `202` — the connection overhead can easily exceed the request itself. Configure retry and backoff on the `ConnectionPool` at initialization instead.
